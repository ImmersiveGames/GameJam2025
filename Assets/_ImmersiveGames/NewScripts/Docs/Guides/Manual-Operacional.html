Como usar os modulos centrais em producao hoje

O estado atual continua o mesmo: startup no bootstrap, frontend e gameplay em SceneRouteKind, navigation e transition por direct-ref + fail-fast, IntroStage level-owned e opcional, PostGame global com Victory, Defeat e Exit, e Restart fora do post hook.

Use istoILevelFlowRuntimeService para abrir gameplay e trocar level local.
Use istoIGameNavigationService para menu e navegacao macro.
Use istoIGameCommands e IPostLevelActionsService para comandos da run e acoes do contexto atual.
Use istoIActorGroupRearmOrchestrator para rearm local de atores.

Receita: criar uma rota nova do zero

O que precisa existir
  • SceneRouteDefinitionAsset
  • SceneKeyAsset das cenas da rota
  • LevelCollectionAsset se a rota for Gameplay
O que configurar
  1. Criar SceneRouteDefinitionAsset.
  2. Preencher routeId, cenas e targetActiveSceneKey.
  3. Escolher routeKind.
  4. Se for gameplay, usar requiresWorldReset=true e apontar levelCollection.
  5. Referenciar a rota por routeRef no catálogo fino de navigation.
Erro comum
Configurar rota Frontend com levelCollection ou rota Gameplay sem requiresWorldReset=true.

Receita: criar um TransitionStyleAsset do zero

O que precisa existir
  • SceneTransitionProfile
  • Path coerente em Assets/Resources/SceneFlow/Styles/
O que configurar
  1. Criar o asset.
  2. Preencher profileRef.
  3. Escolher useFade.
  4. Dar nome estavel e legivel ao style.
Evite isto
Usar o nome do style como semantica de fluxo. Hoje a semantica vem de bootstrap e RouteKind, nao do nome do style.

Receita: ligar o style numa rota

Passo a passo
  1. Abrir GameNavigationCatalogAsset.
  2. Preencher o slot core ou a entry extra com routeRef e transitionStyleRef.
  3. Confirmar que ambos apontam para assets validos.
await navigation.GoToMenuAsync("Frontend/CloseModal");
Erro comum
Deixar transitionStyleRef vazio. O catalogo nao trata isso como opcional.

Receita: configurar startupTransitionStyleRef

O que precisa existir
  • Assets/Resources/NewScriptsBootstrapConfig.asset
  • um TransitionStyleAsset valido
O que configurar
  1. Preencher startupTransitionStyleRef.
  2. Confirmar profileRef valido no style.
  3. Manter startup no bootstrap, nao em route.

Receita: criar um level novo do zero

O que precisa existir
  • SceneBuildIndexRef das cenas do level
  • LevelDefinitionAsset
O que configurar
  1. Preencher additiveScenes.
  2. Ajustar hasIntroStage.
  3. Ajustar hasPostGameReactionHook.
  4. Ajustar cortinas locais se o level precisar.
Erro comum
Criar LevelDefinitionAsset com additiveScenes vazio. Hoje isso invalida o asset.

Receita: colocar o level numa LevelCollection

Passo a passo
  1. Abrir LevelCollectionAsset.
  2. Adicionar os LevelDefinitionAsset em levels.
  3. Deixar o default em levels[0].
  4. Nao repetir referencias.
O que esperar
levels[0] vira o default usado por LevelMacroPrepareService, e a colecao passa a ser a fonte unica de levels da rota de gameplay.

Receita: adicionar uma IntroStage ao level

O que configurar
  1. No LevelDefinitionAsset, ligar hasIntroStage.
  2. Garantir que o level esta na LevelCollectionAsset da rota atual.
  3. Usar o mock atual apenas para testes de desenvolvimento.
O que esperar
  • LevelEnteredEvent dispara a intro depois do level aplicado.
  • LevelIntroCompletedEvent libera o handoff para Playing.
  • Se nao houver intro, o fluxo segue sem erro.
await levelFlow.StartGameplayDefaultAsync("Menu/PlayButton", cancellationToken);
Erro comum
Esperar intro global ou por rota. Hoje ela e level-owned e opcional por LevelDefinitionAsset.

Receita: adicionar hook opcional de post ao level

O que configurar
  1. Ligar hasPostGameReactionHook no LevelDefinitionAsset.
  2. Manter o owner do pos-run no fluxo global.
  3. Usar esse contrato apenas para complementar Victory, Defeat ou Exit.
O que esperar
  • PostGame continua global.
  • O level atual pode complementar com feedback visual.
  • Restart fica fora desse hook.
Evite isto
Tratar esse hook como owner do PostStage. O owner de PostStage e global em Modules/PostGame, nao por level.

Receita: fazer um ator participar do ActorGroupRearm

O que precisa existir
  • ator na cena de gameplay
  • IActorGroupRearmable ou IActorGroupRearmableSync
  • IActorKindProvider quando o alvo for por grupo
Use isto
Comece por ActorGroupRearmRequest.ByActorKind(...). So use ActorIdSet quando o caso realmente exigir ids explicitos.
public sealed class SampleEnemyRearm : MonoBehaviour, IActorGroupRearmable
{
    public Task ResetCleanupAsync(ActorGroupRearmContext ctx) => Task.CompletedTask;
    public Task ResetRestoreAsync(ActorGroupRearmContext ctx) => Task.CompletedTask;
    public Task ResetRebindAsync(ActorGroupRearmContext ctx) => Task.CompletedTask;
}

ActorGroupRearmRequest request = ActorGroupRearmRequest.ByActorKind(ActorKind.Player, "Gameplay/SoftReset");
bool applied = await actorGroupRearm.RequestResetAsync(request);
Erro comum
Usar ActorIdSet como primeira escolha. Hoje o trilho principal e ByActorKind.

Disparar gameplay

O que precisa existir
Bootstrap valido, catalogos resolvidos e rota de gameplay com levelCollection.
using System.Threading;
using System.Threading.Tasks;
using _ImmersiveGames.NewScripts.Modules.LevelFlow.Runtime;

public async Task OnPlayPressedAsync(ILevelFlowRuntimeService levelFlow, CancellationToken cancellationToken)
{
    await levelFlow.StartGameplayDefaultAsync("Menu/PlayButton", cancellationToken);
}
O que esperar
SceneFlow entra na rota de gameplay, o level default e preparado, a intro opcional roda se existir e a run chega em Playing.

Disparar restart

gameCommands.RequestRestart("Pause/RestartButton");

await postLevelActions.RestartLevelAsync("PostGame/RestartButton", cancellationToken);
Use isto
Use IGameCommands para UI simples ou IPostLevelActionsService no contexto atual.
Evite isto
Esperar que restart passe por post hook. Hoje ele vai direto para reset/restart.

Disparar exit to menu

await navigation.ExitToMenuAsync("Pause/ExitToMenu");

await postLevelActions.ExitToMenuAsync("PostGame/ExitToMenu", cancellationToken);
O que esperar
O fluxo publica a intencao de exit, o PostGame global formaliza Exit quando aplicavel e o level atual so pode complementar isso por hook opcional.

Disparar next level

await postLevelActions.NextLevelAsync("PostGame/NextLevel", cancellationToken);
O que esperar
O proximo level sai da LevelCollectionAsset atual e a troca segue pelo trilho local do level atual.

Disparar swap local

await levelFlow.SwapLevelLocalAsync(levelRef, "UI/SelectLevel", cancellationToken);
O que esperar
LevelSelectedEvent abre o trilho, o reset local do level alvo acontece e LevelSwapLocalAppliedEvent fecha a troca local.

Loading de producao

Use isto
LoadingHudScene como HUD canonica do macro flow, controlada por ILoadingPresentationService e exibida por LoadingHudService.
Nao use para isso
Nao trate o loading como dono da navegacao, do reset de mundo, do prepare de level ou da timeline de transicao. O pipeline continua em SceneFlow + WorldLifecycle + LevelFlow.
Onde entrastartup, menu -> gameplay, gameplay -> menu e restart macro.
O que mostrabarra de progresso, porcentagem numerica, etapa atual e spinner.
Como calculaprogresso hibrido: carga real de cena quando existe e marcos ponderados no restante.
Fail-fastbinding incompleto, cena ausente ou root invalido falham explicitamente.
Passo a passo
  1. SceneTransitionService continua owner da transicao macro.
  2. LoadingHudOrchestrator observa o inicio e pede a HUD.
  3. ILoadingPresentationService garante a cena pronta e mostra a HUD.
  4. LoadingProgressOrchestrator envia LoadingProgressSnapshot.
  5. A HUD atualiza barra, porcentagem, etapa e spinner.
  6. No fim real do fluxo, a HUD fecha em 100% e some.
Binding obrigatorio
  • Canvas
  • CanvasGroup
  • loadingText
  • progressPercentText
  • progressFillImage
  • spinnerVisual
  • spinnerTransform
Owner do fluxo
SceneTransitionService, WorldLifecycle e LevelFlow continuam decidindo route transition, reset, prepare e gates.
Owner da apresentacao
ILoadingPresentationService, LoadingHudService e LoadingHudScene apenas apresentam o estado atual do carregamento.
Erro comum
Esperar porcentagem puramente temporal ou usar o loading para esconder problema de pipeline. O contrato atual usa progresso hibrido e mantem a HUD como apresentacao apenas.

Erros comuns por assunto

NavigationNao mexa em asset para simular runtime. Configure no editor e chame service publico com reason clara.
StylesNao trate nome de style como semantica. O owner estrutural continua sendo TransitionStyleAsset.
LevelNao deixe additiveScenes vazio e nao esqueça de colocar o level na LevelCollection.
IntroNao espere intro global. Hoje ela e level-owned e opcional.
Post hookNao transforme o hook opcional em dono do pos-run. O owner continua global.
RearmNao comece por ActorIdSet se ByActorKind resolve o caso.

Checklist de producao

  • bootstrap com navigationCatalog, sceneRouteCatalog, startupTransitionStyleRef e fadeSceneKey
  • GameNavigationCatalogAsset com slots core validos
  • rota Gameplay com levelCollection
  • TransitionStyleAsset com profileRef
  • LevelCollectionAsset com levels ordenados e sem duplicatas
  • LevelDefinitionAsset com additiveScenes validas
  • flags corretas de hasIntroStage e hasPostGameReactionHook
  • atores por grupo com IActorKindProvider quando necessario
  • componentes de rearm com IActorGroupRearmable ou IActorGroupRearmableSync
  • reason clara e rastreavel nas chamadas publicas
