Pooling Canonico - Uso Rapido

Referencia curta para configurar e operar pooling em feature/cena sem bootstrap global por lista.

Fonte tecnica completa: Docs/Guides/Pooling-How-To.md.

O que e

  • Infra compartilhada em Infrastructure/Pooling/**.
  • Identidade estrutural por PoolDefinitionAsset.
  • Ownership global no GlobalCompositionRoot (registro de IPoolService).

Quando usar

  • Objetos de alta rotacao com rent/return repetido.
  • Quando quiser reduzir churn de instantiate/destroy.
  • Quando precisar de auto-return controlado por asset.

Passos rapidos

  1. Criar/configurar PoolDefinitionAsset (prefab, initialSize, canExpand, maxSize, autoReturnSeconds, poolLabel, prewarm).
  2. No consumer, usar a base reutilizavel de dependencias de pool (PoolConsumerBehaviourBase, namespace Infrastructure.Pooling.Interop).
  3. Configurar lista explicita de pools no consumer.
  4. Deixar a base aplicar automaticamente:
    • EnsureRegistered para todos
    • Prewarm apenas para assets com prewarm=true
  5. Usar Rent / Return no fluxo da feature.

Checklist curto

Pitfalls comuns

  • Nao usar string como chave estrutural de pool.
  • Nao colocar regra de dominio dentro do pooling.
  • Nao marcar prewarm=true sem necessidade real de aquecimento.
  • Nao ignorar limite maximo em cenarios de burst.

Auto-return (resumo)

  • autoReturnSeconds <= 0: desativado.
  • autoReturnSeconds > 0: retorno automatico por instancia rented.
  • Return manual e cleanup cancelam timers corretamente.
